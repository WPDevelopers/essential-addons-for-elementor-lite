# Content Ticker Widget

> Single-row news/headline ticker built on Swiper — pulls posts via `WP_Query` (or Pro-only custom Repeater rows) and rotates them horizontally with fade or slide effect, a fixed badge label ("Trending Today") to the left, and prev/next arrows. Uses Elementor's `e-swiper` style handle and the `elementorFrontend.utils.swiper` async loader; falls back to a `new Swiper()` constructor when Swiper is already loaded.

**Class file:** [`includes/Elements/Content_Ticker.php`](../../includes/Elements/Content_Ticker.php)
**Slug:** `content-ticker` (widget id `eael-content-ticker`)
**Public docs:** <https://essential-addons.com/elementor/docs/content-ticker/>
**Pro-shared:** ✅ Yes — Pro adds the `custom` ticker type via `apply_filters('eael_ticker_options', …)` (adds "Custom" to the chooser) and `do_action('eael_ticker_custom_content_controls', $this)` (injects the custom-content Repeater section during `register_controls`). Pro can also drop additional templates into `essential-addons-elementor/includes/Template/Content-Ticker/` for the template dropdown.

---

## Overview

Content Ticker renders a horizontal Swiper carousel of "headline" slides — typically one slide visible at a time scrolling left or right at a configurable interval — preceded by a static badge text. Two content sources: `dynamic` runs a `WP_Query` (post type / taxonomy / orderby — injected via the internal `eael/controls/query` action) and includes a template file per post; `custom` (Pro-only) uses a Repeater of text + link entries. Slide markup is generated by template files under `includes/Template/Content-Ticker/` — Lite ships a single `default.php` template (Template Name: `Default`); Pro and themes can register additional templates that appear in the Template Layout dropdown. The Swiper is initialized via `elementorFrontend.utils.swiper` (async loader) when Swiper is not yet available, or via direct `new Swiper()` when it is. Right-to-left scrolling is achieved by setting `dir="rtl"` on the swiper container (a content-ticker-specific trick — not Elementor's site-wide RTL).

## Architecture

- **No bundled Swiper — relies on Elementor's handle and async loader** — `get_style_depends()` returns `['e-swiper']` (Elementor's Swiper 8 CSS handle). The widget never bundles its own Swiper. JS calls `elementorFrontend.utils.swiper` (returns a Promise that lazily loads Swiper) when `typeof Swiper === 'undefined' || === 'function'` ([JS line 149-153](../../src/js/view/content-ticker.js#L149)), otherwise wraps `new Swiper(el, config)` in a Promise. The `'function'` check is suspicious — `typeof Swiper === 'function'` is true when Swiper IS loaded as a class, which means this branch ALWAYS uses the async loader. Effectively `new Swiper()` direct path is dead code unless Swiper is loaded as an object literal. ⚠
- **Two ticker types in one render branch** — `dynamic` runs `WP_Query` with args from `Helper::get_query_args($settings)` and `include`s the template inside the `while ($query->have_posts())` loop ([line 783-799](../../includes/Elements/Content_Ticker.php#L783)). `custom` (Pro-only — gated by `apply_filters('eael/is_plugin_active', '<plugin-path>')`) iterates `$settings['eael_ticker_custom_contents']` and calls `Helper::include_with_variable($template, ['content' => …, 'link' => …])` which `extract()`s vars into the template scope before including. The template at `default.php` checks `isset($content) && isset($link)` to branch between the two modes ([template line 12](../../includes/Template/Content-Ticker/default.php#L12)).
- **Tribe Events monkey-patch** — when `$args['post_type'] === 'tribe_events'`, calls `$this->eael_forcefully_remove_action('pre_get_posts', 'attach_monitor', 200)` ([line 787](../../includes/Elements/Content_Ticker.php#L787)). Tribe's Event Calendar attaches a `pre_get_posts` handler that mutates queries; the widget yanks it before `new WP_Query()` to keep its own query args intact. Hard-coded compatibility shim for a single 3rd-party plugin.
- **Right-to-left direction via `dir="rtl"` attribute** — when `direction == 'right'`, render adds `dir="rtl"` to the swiper container ([line 738-740](../../includes/Elements/Content_Ticker.php#L738)). This is a Swiper trick: the framework reads the attribute and reverses slide order. Not the same as Elementor's site-wide `body.rtl` class.
- **Autoplay-off via cosmic `999999` delay** — when `autoplay != 'yes'`, render writes `data-autoplay="999999"` ([line 757](../../includes/Elements/Content_Ticker.php#L757)). JS passes this as `autoplay.delay` (~16.7 minutes). `autoPlayManager()` separately handles the case `options.autoplay.delay === 0` by calling `event.autoplay.stop()` — but the off path uses the huge number, not zero. Effectively autoplay never stops, just yields ridiculously slow.
- **Pagination `.swiper-pagination-<id>` selector is configured but never rendered** — `data-pagination` is set on the swiper element ([line 734](../../includes/Elements/Content_Ticker.php#L734)) and JS uses it in Swiper config, but `render()` and `render_arrows()` emit only the prev/next buttons inside `.content-ticker-pagination`, never a `.swiper-pagination` element. Swiper silently no-ops when its pagination target doesn't exist. ⚠ Dead config.
- **JS reads `data-items` / `data-items-tablet` / `data-items-mobile` / `data-margin` that PHP never sets** — ([JS line 3-26](../../src/js/view/content-ticker.js#L3)) — defaults to `1` for items and `10` for margin when undefined. PHP only sets `data-margin-tablet` and `data-margin-mobile` (when those controls have non-empty sizes). The unused data-attribute branches are dead code paths kept for forward-compatibility. Effectively `slidesPerView: 1` always.
- **Cross-widget reflow destroys and recreates the Swiper instance** — `ContentTickerSlider` callback ([JS line 126-140](../../src/js/view/content-ticker.js#L126)) finds each `.eael-content-ticker`, calls `$this[0].swiper.destroy(true, true)`, then re-runs `swiperLoader()` with fresh config. Subscribes to `ea-toggle-triggered`, `ea-lightbox-triggered`, `ea-advanced-tabs-triggered`, `ea-advanced-accordion-triggered`. More aggressive than Filterable_Gallery (just relayouts) or Adv_Tabs (just `update()` calls) — Content_Ticker rebuilds from scratch because Swiper's measurements drift hard when the container was hidden.
- **`get_template_list_for_dropdown()` populates the Template Layout dropdown** at [line 90](../../includes/Elements/Content_Ticker.php#L90) by scanning three locations: `includes/Template/Content-Ticker/` in Lite, the same in Pro (when active), and `<theme>/Template/Content-Ticker/` in the active theme. Each PHP file with a `Template Name:` header becomes a dropdown entry. Theme-override path lets sites customize ticker markup without touching the plugin.
- **Control id has a stray capital** — `eael_dynamic_template_Layout` (capital `L` on `Layout`) is the dropdown control id ([line 85](../../includes/Elements/Content_Ticker.php#L85)). Stable for backwards compat; cosmetic only.

```html
<div class="eael-ticker-wrap" id="eael-ticker-wrap-<widget-id>">

  [?] <!-- Badge: visible when eael_ticker_tag_text is non-empty -->
  <div class="ticker-badge">
    <span>Trending Today</span>
  </div>

  <div class="eael-ticker">
    <div class="swiper swiper-8 eael-content-ticker swiper-container-<widget-id>"
         data-pagination=".swiper-pagination-<widget-id>"        ← target doesn't exist; dead config
         data-arrow-next=".swiper-button-next-<widget-id>"
         data-arrow-prev=".swiper-button-prev-<widget-id>"
         [?] dir="rtl"                                            ← when direction == 'right'
         [?] data-margin-tablet="N"
         [?] data-margin-mobile="N"
         data-effect="slide | fade"
         data-speed="400"
         data-autoplay="2000 | 999999"                            ← 999999 = autoplay off
         [?] data-pause-on-hover="true"
         [?] data-loop="true"
         [?] data-grab-cursor="true"
         [?] data-arrows="1">

      <div class="swiper-wrapper">
        <!-- For dynamic mode: per-post template include -->
        <!-- For custom mode (Pro): per-Repeater-item template include -->
        <!-- Default template emits: -->
        <div class="swiper-slide">
          <div class="ticker-content">
            [?] <a href="…" [target="_blank"] [rel="nofollow"]>
                  {wp_kses content OR get_the_title()}
                </a>
            [or just plain content / title text when no link]
          </div>
        </div>
        …
      </div>
    </div>

    [?] <!-- Arrows wrapper: visible when arrows == 'yes' -->
    <div class="content-ticker-pagination">
      <div class="swiper-button-next swiper-button-next-<widget-id>">
        <i class="<arrow-icon>"></i>     <!-- or <img> for SVG icon library -->
      </div>
      <div class="swiper-button-prev swiper-button-prev-<widget-id>">
        <i class="<prev-arrow-icon>"></i>
      </div>
    </div>
  </div>
</div>
```

Notes:

- `swiper-8` class is a legacy compatibility marker for Elementor's Swiper 8 migration — does not currently affect behavior but is widely keyed off in old EA CSS.
- `.swiper-slide.swiper-slide-prev` and `.swiper-slide-next` have `opacity: 0 !important` in SCSS ([line 34-38](../../src/css/view/content-ticker.scss#L34)) — keeps only the centred slide visible. This is the visual mechanism that makes the ticker "rotate one item at a time" even when `slidesPerView` would technically show neighbours during transitions.
- Empty-state message `<div class="swiper-slide"><a href="#" class="ticker-content">No content found!</a></div>` rendered when `WP_Query::have_posts()` returns false ([line 798](../../includes/Elements/Content_Ticker.php#L798)).
- Mobile (`max-width: 767px`) hides `.content-ticker-pagination` entirely via SCSS ([line 60-62](../../src/css/view/content-ticker.scss#L60)) — no arrows on phones.

## JavaScript Lifecycle

- **Trigger:** `eael.hooks.addAction("init", "ea", () => elementorFrontend.hooks.addAction('frontend/element_ready/eael-content-ticker.default', ContentTicker))` — newer EA registration pattern.
- **Guard:** none — no `elementStatusCheck` flag. Re-fires of `frontend/element_ready` re-init.
- **Vendor dependency:** Swiper 8 (loaded via `elementorFrontend.utils.swiper`, NOT bundled).
- **Reads on init:** all 13 `data-*` attributes from `.eael-content-ticker`: `items`, `items-tablet`, `items-mobile`, `margin`, `margin-tablet`, `margin-mobile`, `effect`, `speed`, `autoplay`, `loop`, `grab-cursor`, `pagination` (selector), `arrow-next`, `arrow-prev`, `pause-on-hover`. (Note: `items*` and `margin` (no breakpoint) are never set by PHP — defaults to 1 / 10 from JS.)
- **Branches:**
  - if `Swiper` is undefined OR a function — use `elementorFrontend.utils.swiper` async loader (returns Promise).
  - else — wrap `new Swiper(el, config)` in a Promise.
  - `autoPlayManager` post-init — if `autoplay.delay === 0`, calls `event.autoplay.stop()`; if `pauseOnHover` and delay non-zero, binds mouseenter (pause) / mouseleave (run).
- **Cross-widget reflow:** `ContentTickerSlider` callback handles all four EA events. For each `.eael-content-ticker` in the activated element: `swiper.destroy(true, true)` → re-read settings → re-init via `swiperLoader()`. Destroys are necessary because Swiper's `update()` doesn't fully recompute on a previously-hidden container; rebuild is the brute-force fix.
- **No keyboard a11y:** Swiper's default keyboard module is not enabled in config.

## Common Issues

### Ticker shows no slides / "No content found!"

- **Likely cause:** `WP_Query` with the active args has zero matching posts. Common with overly-restrictive taxonomy or date filters.
- **Diagnose:** dump `$args = Helper::get_query_args($settings)` and run `new WP_Query($args)` manually. Check `post_status`, `post_type`, taxonomy terms.
- **Fix:** broaden the query in the panel; verify selected `post_type` matches the template's expectations.

### Ticker scrolls in the wrong direction even after toggling Direction

- **Likely cause:** the page's body has `body.rtl` class (site is RTL), AND `direction == 'right'` adds another `dir="rtl"` on the swiper container — they cancel out.
- **Diagnose:** check `<body>` for `rtl` class; check the swiper container's `dir` attribute.
- **Fix:** on RTL sites, set Direction to `left` to get visually-right scrolling. The widget doesn't auto-account for site RTL.

### Autoplay doesn't stop when "Autoplay" toggle is off

- **Likely cause:** "off" sets `data-autoplay="999999"` (~16.7 minutes), not `0`. Strictly speaking, the ticker still autoplays — just imperceptibly slowly.
- **Diagnose:** inspect `data-autoplay` on the swiper container.
- **Fix:** working as designed. To truly stop autoplay, manually set the data attribute to `0` post-render or edit JS to special-case `999999` as "stop".

### Tribe Events queries return wrong posts

- **Likely cause:** Tribe's `attach_monitor` hook at priority 200 on `pre_get_posts` was supposed to be removed, but the helper `eael_forcefully_remove_action()` failed (e.g., Tribe registered the hook on a different class or after the widget's render).
- **Diagnose:** dump `$GLOBALS['wp_filter']['pre_get_posts']` before `new WP_Query($args)`.
- **Fix:** widget already handles this in render; if it still misbehaves, ensure EA's `eael_forcefully_remove_action()` trait is loaded and the Tribe class name matches the hardcoded value in the helper.

### Ticker is stuck / collapsed after opening it in a tab or accordion

- **Likely cause:** Swiper measured the container at zero width while it was hidden; the cross-widget reflow listener didn't fire because the parent widget doesn't broadcast the expected event.
- **Diagnose:** confirm that the parent (Adv_Tabs / Adv_Accordion / Toggle / Lightbox) is the EA Pro version that broadcasts `ea-<widget>-triggered`.
- **Fix:** working as intended for EA-wrapped containers. Third-party tabs / accordions need to broadcast one of `eael.hooks.doAction("ea-advanced-tabs-triggered", $container)` etc. from their own JS for the rebuild to fire.

### Custom template doesn't appear in the Template Layout dropdown

- **Likely cause:** the template file is missing the `Template Name:` header comment, OR it's in the wrong directory.
- **Diagnose:** template files must live in `<theme>/Template/Content-Ticker/<file>.php` and start with a `/** Template Name: My Layout */` docblock.
- **Fix:** add the docblock; dropdown re-populates on next page load (no cache to clear).

## Known Limitations

- **`new Swiper()` direct path is effectively dead** — the `typeof Swiper === 'function'` guard ([JS line 149](../../src/js/view/content-ticker.js#L149)) treats "Swiper is loaded as a class" the same as "undefined" and routes both to the async loader. The Promise-wrapping `swiperPromise()` function is unreachable code unless Swiper is loaded as an `object` somehow. Cosmetic but confusing.
- **`data-items` / `data-items-tablet` / `data-items-mobile` / `data-margin` (un-breakpointed) are read by JS but never written by PHP** — the JS defaults handle this gracefully (`slidesPerView: 1`, `spaceBetween: 10`), but the control surface for per-breakpoint items count is missing entirely. A panel "Slides Per View" control would not work today without a render change too.
- **Two un-prefixed legacy hooks** — `eael_ticker_options` and `eael_ticker_custom_content_controls` predate the `eael/<context>/<action>` naming convention. Renaming requires dual-emit migration.
- **Dead pagination configuration** — `data-pagination=".swiper-pagination-<id>"` is set, JS passes it to Swiper config, but the element is never rendered. Swiper silently no-ops. Could become a real pagination if `render_arrows()` (or a sibling method) emits `.swiper-pagination` — but currently a configured-but-disabled feature.
- **Autoplay "off" uses `999999` magic number** — not a clean off state; effectively a 16-minute period. `autoPlayManager` has a `delay === 0` branch but the off path never hits it.
- **Tribe Events compat is hardcoded** — single hardcoded plugin compat at [line 786-788](../../includes/Elements/Content_Ticker.php#L786). Other date/event plugins that hook `pre_get_posts` (e.g., The Events Calendar Pro, Modern Events Calendar) are not handled.
- **Mobile (≤767px) hides arrows unconditionally** — `.content-ticker-pagination { display: none }` in SCSS. No control to override.
- **Control id has stray capital** — `eael_dynamic_template_Layout` (capital `L`). Stable for backwards compat; renaming would break saved widget data.
- **No `is_dynamic_content()` override that disables cache** — render contains saved-template includes (which can be dynamic), but the widget doesn't disable Elementor's render cache for saved-template content the way Adv_Accordion and Adv_Tabs do. Edits to ticker templates may not appear until the page cache expires.
- **Pause-on-hover JS binds events on every init** — re-fires of `frontend/element_ready` re-bind `mouseenter` / `mouseleave` handlers without unbinding, accumulating handlers across re-inits.
- **No `role="region"` / `aria-live="polite"`** on the ticker — for a rotating news headline, screen readers should announce new slides. Missing announcement attributes.
