# Woo Product Carousel Widget

> Swiper-driven WooCommerce product carousel with 4 preset templates, coverflow / slide effects, autoplay + marquee modes, full per-device responsive controls, and Quick View popup. Uses Elementor's `e-swiper` handle (no bundled Swiper) plus `elementorFrontend.utils.swiper` async loader. Pro extends via auto-merged template directory + a GSAP marquee variant; Lite ships a Swiper-based marquee fallback (`no-gsap`).

**Class file:** [`includes/Elements/Woo_Product_Carousel.php`](../../includes/Elements/Woo_Product_Carousel.php)
**Slug:** `woo-product-carousel` (widget id `eael-woo-product-carousel`) — slug + widget id consistent.
**Public docs:** <https://essential-addons.com/elementor/docs/woo-product-carousel/>
**Pro-shared:** ✅ Yes — Pro adds additional preset templates via `Template_Query` auto-merge (`EAEL_PRO_PLUGIN_PATH/includes/Template/Woo-Product-Carousel/`), and Pro path of marquee mode (GSAP + Draggable + eaelmarque jQuery plugin). Pro detection via `eael/pro_enabled` filter; no `eael_section_pro` upsell panel.

---

## Overview

Woo Product Carousel renders a WP_Query-driven WooCommerce product carousel. Each rendered slide is generated by `include`-ing the preset's PHP template inside the loop ([line 3361-3364](../../includes/Elements/Woo_Product_Carousel.php#L3361)) — the same template-include pattern as Product_Grid. Four Lite presets ship in [`includes/Template/Woo-Product-Carousel/`](../../includes/Template/Woo-Product-Carousel/) (preset-1..preset-4); Pro extends by dropping more PHP templates with `Template Name:` headers into the parallel Pro directory.

The carousel supports two effects (`slide` + `coverflow`) and a marquee mode gated by `autoplay=yes && carousel_effect=slide && enable_marquee=yes`. Marquee has a Pro path (GSAP + Draggable + `eaelmarque()` jQuery plugin) and a Lite fallback labelled `no-gsap` (Swiper with `loop:true` + `autoplay.delay:0` + `allowTouchMove:false`). JS reads ~15 data attributes from the wrapper and async-loads Swiper via `elementorFrontend.utils.swiper`.

Quick View modal is shared with Product_Grid (loaded via `quick-view.min.js` in `config.php`). Compare button is NOT included in this widget (Pro adds it).

## Architecture

- **Template-include rendering with auto-merged directories** — `render()` calls `get_template($settings['eael_dynamic_template_layout'])` then `include`s the matching file inside `while( $query->have_posts() )` ([line 3361-3364](../../includes/Elements/Woo_Product_Carousel.php#L3361)). `Template_Query::get_template_files()` scans Lite + Pro + active-theme `Template/Woo-Product-Carousel/` dirs and merges them. Adding a new preset = drop PHP file with `/** Template Name: Preset 5 */` header. Same pattern as Product_Grid but different directory name.
- **Marquee gating** ([line 3170, 3204-3206, 3224-3227](../../includes/Elements/Woo_Product_Carousel.php#L3170)) — `$is_marquee = 'slide' === carousel_effect && enable_marquee === 'yes'`. Pro path: `is_marquee && has_pro` adds `.eael-marquee-carousel` class (no `swiper-container-wrap`), JS calls `eaelmarque()` jQuery plugin (GSAP + Draggable). Lite fallback: `is_marquee && !has_pro` adds `.eael-marquee-carousel.no-gsap` + `data-autoplay='1'`, still inside Swiper but with `loop:true`, `autoplay.delay:0` (continuous scroll), `allowTouchMove:false`.
- **Swiper loaded via `elementorFrontend.utils.swiper` async loader** ([line 356-365 of woo-product-carousel.js](../../src/js/view/woo-product-carousel.js#L356)) — `swiperLoader()` helper returns either `new asyncSwiper(...)` Promise OR direct `new Swiper(...)` Promise wrapper. CSS via Elementor's `e-swiper` handle (no bundled fallback in this widget).
- **No `eael_section_pro` upsell** — Pro features (GSAP marquee, extra presets) detected inline via `eael/pro_enabled` filter; no panel-side warning when Pro layout selected (because all 4 Lite presets are Lite-available; Pro just adds more).
- **Quick View popup body-injection on every widget** ([line 299-311 of woo-product-carousel.js](../../src/js/view/woo-product-carousel.js#L299)) — `.eael-woocommerce-popup-view` appended to `<body>` if not present. Single-element check (`length < 1`) prevents duplicate, but the popup is shared with Product_Grid + Woo_Product_List. Last widget to render owns popup state.
- **Cross-widget reflow listeners** ([line 326-329 of woo-product-carousel.js](../../src/js/view/woo-product-carousel.js#L326)) — `WooProductCarouselLoader` destroys + recreates Swiper instance on 4 EA events: `ea-lightbox-triggered`, `ea-toggle-triggered`, `ea-advanced-tabs-triggered`, `ea-advanced-accordion-triggered`. Aggressive re-init (destroy true, true) — saved params re-applied; preserves swiper state by reading `[0].swiper.params`.
- **Coverflow has HARDCODED breakpoints** ([line 238-248 of woo-product-carousel.js](../../src/js/view/woo-product-carousel.js#L238)) — `{1024: {slidesPerView: 3}, 768: 1, 320: 1}` — ignores `items_tablet` / `items_mobile` panel settings entirely. ⚠️ User panel changes won't reflect on coverflow effect.
- **Slide effect has 2 breakpoint paths** — if `localize.el_breakpoints` is a string (legacy), uses hardcoded 1024/768/320 mapping; otherwise iterates Elementor's breakpoint config (`mobile`, `mobile_extra`, `tablet`, `tablet_extra`, `laptop`, `desktop`, `widescreen`) and reads matching `data-items-{device}` / `data-slide-items-{device}` / `data-margin-{device}` from wrapper.
- **`autoplay` magic number 999999** ([line 65 of woo-product-carousel.js](../../src/js/view/woo-product-carousel.js#L65)) — when no `data-autoplay` attr, defaults to `999999` ms (~16.7 min). Same pattern as Content_Ticker. Then guarded by `if ($autoplay === 0) autoplay = false` — but `999999` never equals `0`, so autoplay always runs.
- **`eael_woo_after_product_loop` fired twice** ([line 3367 + 3388](../../includes/Elements/Woo_Product_Carousel.php#L3367)) — once inside `if ($query->have_posts())` after the inner wrapper close, once unconditionally at outer scope. Same double-fire bug as Product_Grid. Listeners must `func_num_args()` or idempotency check.
- **`product_query_builder()` shares logic with Product_Grid** ([line 3635](../../includes/Elements/Woo_Product_Carousel.php#L3635)) — but defined LOCALLY in this class, not via shared trait. Drift between this widget's query logic and Product_Grid's is invisible.
- **Status filter `eael/woo-product-carousel/product-statuses`** ([line 152](../../includes/Elements/Woo_Product_Carousel.php#L152)) — defaults include `publish`, `draft`, `pending`, `future`. Same panel exposure concern as Product_Grid; this widget has no separate `wp_ajax_*` endpoint of its own (uses initial render only — no load-more), so the AJAX-tampering vector doesn't apply.
- **`load_quick_view_asset()` enqueues WC's photoswipe + zoom + flexslider + wc-single-product + wc-add-to-cart-variation on `wp_footer`** — only when WC supports them (`current_theme_supports('wc-product-gallery-*')`). Same helper used by Product_Grid.

## JavaScript Lifecycle

- **Trigger:** `elementorFrontend.hooks.addAction('frontend/element_ready/eael-woo-product-carousel.default', wooProductCarousel)` ([line 378-381](../../src/js/view/woo-product-carousel.js#L378))

- **Vendor deps:** Elementor's Swiper (via `e-swiper` handle + `elementorFrontend.utils.swiper` async loader); jQuery; `eaelmarque` (Pro-only — defined inside Pro's bundle when GSAP marquee active)
- **Reads on init:** ~15 data attrs from `.eael-woo-product-carousel` first child: `pause-on-hover`, `speed`, `margin` (+ per-device), `autoplay`, `pagination`, `arrow-next/prev`, `items` (+ per-device), `slide-items` (+ per-device), `effect`, `loop`, `grab-cursor`, `depth`, `rotate`, `stretch`
- **Marquee branching:** if `hasClass('eael-marquee-carousel') && !hasClass('no-gsap')` → Pro path: `initMarquee()` calls `$wooProductCarousel.find('.eael-marquee-wrapper').eaelmarque({speed, direction:'left', pauseOnHover, draggable})`; window resize listener calls `updateMarquee()` to recompute per-item width via `(100 / itemsPerView) + '%'`. Otherwise → Swiper path.
- **Swiper-init via `swiperLoader()` helper** ([line 356-365](../../src/js/view/woo-product-carousel.js#L356)) — uses `elementorFrontend.utils.swiper` async loader when Swiper not yet defined; otherwise direct `new Swiper(...)` wrapped in Promise.
- **Marquee Lite (`no-gsap`) Swiper config:** `loop: true` (forced), `autoplay.delay: 0` (continuous), `allowTouchMove: false` (no drag), uses regular Swiper transitions — visually approximates marquee but isn't true continuous scroll.
- **Coverflow hardcoded breakpoints** ([line 238-248](../../src/js/view/woo-product-carousel.js#L238)) — ignores `items_tablet` / `items_mobile`; always 3 desktop / 1 tablet / 1 mobile.
- **Pagination gallery (image dots)** — when `.eael-woo-product-carousel-gallary-pagination` exists, second Swiper instance initialized with `slideToClickedSlide:true`, `slidesPerView:3`, then linked via `eaelWooProductCarousel.controller.control = $paginationGallerySlider` (bidirectional sync).
- **`isEditMode` patch** — forces `.eael-product-image-wrap .woocommerce-product-gallery` opacity to 1 (WC hides product gallery in non-front contexts).
- **Body popup wrapper appended once** ([line 299-311](../../src/js/view/woo-product-carousel.js#L299)) — checked via `length < 1`; same pattern as Product_Grid. Multi-Woo-widget pages share the popup.
- **Cross-widget reflow** ([line 326-329](../../src/js/view/woo-product-carousel.js#L326)) — `WooProductCarouselLoader` destroys (`destroy(true, true)`) and rebuilds Swiper with saved params on 4 EA events (lightbox / toggle / advanced-tabs / advanced-accordion triggered). Aggressive — full re-init each time vs. just calling `.update()`.

## Common Issues

### Marquee feels jumpy on Lite (vs smooth on Pro)

- **Likely cause:** Lite fallback is Swiper-based (`no-gsap` path) using `loop: true` + `autoplay.delay: 0`. Swiper transitions are discrete slide animations; can't approximate continuous-scroll GSAP marquee.
- **Diagnose:** check JS console — does `eaelmarque is not a function` appear? That confirms Lite fallback.
- **Fix:** activate EA Pro for GSAP path; or accept the Lite UX trade-off

### Coverflow shows wrong number of items on tablet/mobile

- **Likely cause:** [line 238-248 of woo-product-carousel.js](../../src/js/view/woo-product-carousel.js#L238) — coverflow has HARDCODED breakpoints `{1024: 3, 768: 1, 320: 1}`; panel-side `items_tablet`/`items_mobile` are ignored
- **Diagnose:** inspect data attrs vs Swiper instance config
- **Fix:** known limitation; switch to `slide` effect for full responsive control

### Carousel doesn't update after closing a lightbox or switching a tab

- **Likely cause:** Cross-widget reflow listener fires `destroy(true, true)` then re-init — if saved params have closures or DOM refs, re-init can fail silently
- **Diagnose:** browser console for Swiper errors after triggering one of: lightbox close, toggle, advanced-tabs switch, advanced-accordion expand
- **Fix:** known aggressive re-init pattern; report if it persistently breaks. Workaround: avoid placing this widget inside accordion/tab/toggle/lightbox content

### Custom preset doesn't appear in layout picker

- **Likely cause:** `Template_Query::get_template_files()` scans `includes/Template/Woo-Product-Carousel/` (process_directory_name converts `eael-woo-product-carousel` → `Woo-Product-Carousel`). Theme/Pro must use this exact directory casing.
- **Diagnose:** verify template file is at `wp-content/plugins/<plugin>/includes/Template/Woo-Product-Carousel/<name>.php` with `/** Template Name: <Name> */` header
- **Fix:** rename directory + add `Template Name:` header

### Quick View popup state shared across multiple Woo widgets

- **Likely cause:** `.eael-woocommerce-popup-view` is appended to `<body>` once per page (first widget); subsequent widgets reuse. Compare modal (Product_Grid's) ALSO renders to body. State conflicts possible if two widgets compete.
- **Diagnose:** inspect body for `.eael-woocommerce-popup-view` count
- **Fix:** known design — shared popup is intentional but state isn't widget-id scoped

### Autoplay doesn't stop after `autoplay` switcher disabled

- **Likely cause:** JS reads `data-autoplay` attr — if `autoplay=no` panel-side, PHP emits `data-autoplay="0"` ([line 3291](../../includes/Elements/Woo_Product_Carousel.php#L3291)); JS at [line 251-253](../../src/js/view/woo-product-carousel.js#L251) checks `if ($autoplay === 0) autoplay = false` (strict equality). If browser parses `data-autoplay="0"` as `"0"` (string) instead of `0` (number), check fails — autoplay enabled with `delay: "0"`.
- **Diagnose:** browser inspect — what's `$autoplay` value in DevTools console?
- **Fix:** strict equality check needs `=== "0"` OR `$autoplay = parseInt($autoplay, 10)`; known fragility

## Known Limitations

- **`eael_woo_after_product_loop` fires twice per render** ([line 3367 + 3388](../../includes/Elements/Woo_Product_Carousel.php#L3367)) — once inside `have_posts()` branch, once unconditionally outside. Same double-fire as Product_Grid. Listeners must be idempotent.
- **`eael_woo_before_product_loop` emitted no-arg** ([line 3352](../../includes/Elements/Woo_Product_Carousel.php#L3352)) — Product_Grid passes `$preset_slug` as first arg ([Product_Grid line 3712](../../includes/Elements/Product_Grid.php#L3712)). Signature inconsistency between two widgets that fire the same hook name.
- **`product_query_builder()` locally defined** ([line 3635](../../includes/Elements/Woo_Product_Carousel.php#L3635)) — not via shared trait with Product_Grid. Logic drift between two widgets that should behave identically is invisible.
- **Coverflow effect hardcodes responsive breakpoints** ([line 238-248 of woo-product-carousel.js](../../src/js/view/woo-product-carousel.js#L238)) — 1024 → 3 items, 768 → 1, 320 → 1; panel-side `items_tablet`/`items_mobile` controls ignored on coverflow.
- **`empty()` function `eael_woo_product_carousel_layout()`** ([line 209-211](../../src/js/view/woo-product-carousel.php#L209)) — declared but body is empty. Dead code, likely placeholder for Pro override pattern (Pro can swap by overriding).
- **`get_style_depends() = ['font-awesome-4-shim']` and `get_script_depends() = ['font-awesome-4-shim']`** — both reference the deprecated `font-awesome-4-shim` handle. Modern Elementor versions may not register this handle, causing enqueue warnings.
- **Marquee Pro path requires GSAP + Draggable + `eaelmarque` jQuery plugin** — none of which are loaded in Lite (config.php has commented-out entries for them). Pro must register these or marquee silently degrades to Swiper fallback.
- **Status allowlist exposes draft / pending / future via panel** ([line 152](../../includes/Elements/Woo_Product_Carousel.php#L152)) — same risk surface as Product_Grid; this widget has no AJAX endpoint (no load-more), so tampering vector is narrower (server-side render only).
- **Body popup wrapper shared across Woo widgets** — `.eael-woocommerce-popup-view` injected to `<body>` once per page; if both Woo_Product_Carousel and Product_Grid render, last to render owns popup. Multi-instance leak.
- **Autoplay default magic-number `999999`** ([line 65 of woo-product-carousel.js](../../src/js/view/woo-product-carousel.js#L65)) — same pattern as Content_Ticker. Effectively autoplay-enabled even when panel says "no" if data-attr missing.
- **Cross-widget reflow uses `destroy(true, true)` + full re-init** — heavier than necessary; `.update()` would suffice for most cases. Re-init may lose user's manual slide position state.
- **Image-dots Swiper pagination has bidirectional controller link without lifecycle teardown** — if main carousel destroys/rebuilds, the pagination Swiper's `controller.control` reference dangles.
- **JS reads `localize.el_breakpoints` and may receive string** ([line 154 of woo-product-carousel.js](../../src/js/view/woo-product-carousel.js#L154)) — defensive `typeof === 'string'` check exists with legacy hardcoded breakpoint path. Dual-path maintenance burden.
- **No `elementStatusCheck` on marquee init path** — only main `wooProductCarousel` has the guard; if marquee branch is hit twice (re-init), `eaelmarque()` plugin may double-bind.
- **`enable_marquee` requires `autoplay=yes`** — sensible default, but users disabling autoplay without disabling marquee may see broken state (Pro JS branches early without marquee init if not autoplay).
- **No `wpml_object_id` filter** — image / template id usage doesn't pass through WPML translation. Compare to [`_patterns.md § WPML`](_patterns.md#wpml-media-translation).
