# Post Grid Widget

> Renders a WP_Query-driven grid of posts with multiple skin templates (preset layouts), Isotope-powered masonry layout, Load More button or infinity scroll, and shared query / button-style controls injected via Bootstrap action hooks. Templates are PHP files included inside the post loop — adding a new skin means dropping a template file.

**Class file:** [`includes/Elements/Post_Grid.php`](../../includes/Elements/Post_Grid.php)
**Slug:** `post-grid` (widget id `eael-post-grid`)
**Public docs:** <https://essential-addons.com/elementor/docs/post-grid/>
**Pro-shared:** ❌ No widget-specific Pro extension — Pro ships a separate `Post_Carousel` widget that shares the `eael-post-grid` CSS and template files but is a distinct class. No `do_action` extension hooks specifically for Post_Grid render output. The four `eael/controls/*` actions Lite emits are **internal code reuse** (registered by Lite's own Bootstrap), not Pro hooks.

---

## Overview

Post Grid is one of EA's heaviest Lite widgets — a configurable post listing with WP_Query backend, ten-plus pre-built skin templates (image-on-top, image-on-side, overlay, card variants), responsive column controls, and a Load More / infinity-scroll pagination chain. Each rendered post is generated by `include`-ing the matching template file inside the WP_Query loop, so the same widget can produce visually different output without conditional render branches. Isotope provides the masonry layout option; imagesLoaded coordinates the layout reflow after images settle.

The Load More / infinity-scroll system is shared infrastructure used by multiple EA widgets (Product Grid, Filterable Gallery in Pro, Woo Product List). All trigger the same `load_more` AJAX endpoint in `Ajax_Handler` trait; the handler dispatches by `class` parameter to the right render path.

## Architecture

- **Template-include rendering** — `render()` loops `WP_Query::have_posts()` and `include`s the matching template file (`get_template($preset_style)`) per post ([line 1845-1847](../../includes/Elements/Post_Grid.php#L1845)). Adding a new skin means dropping a PHP file into `includes/Templates/Post-Grid/` — no PHP class changes needed.
- **Four shared control sets injected via internal action hooks** — `do_action('eael/controls/query', $this)`, `do_action('eael/controls/custom_positioning', $this, …)` (fired 3 times), `do_action('eael/controls/read_more_button_style', $this)`, `do_action('eael/controls/load_more_button_style', $this)`. These are NOT Pro extension hooks — they're internal code reuse. Bootstrap [registers all four listeners](../../includes/Classes/Bootstrap.php#L184) within Lite itself. Same control sets are used by Adv_Accordion, Woo_Product_List, etc.
- **Transient hack to pass settings to template files** — `set_transient('eael_post_grid_read_more_button_text_<id>', …, DAY_IN_SECONDS)` and `_excerpt_expanison_indicator_<id>` ([line 1830-1831](../../includes/Elements/Post_Grid.php#L1830)) — template files read these via `get_transient()`. Workaround for the fact that template files run in the WP_Query loop without direct access to widget settings; transients persist for 24h. Typo: `expanison` should be `expansion`.
- **Editor preview emits inline `<script>`** ([line 1866-1894](../../includes/Elements/Post_Grid.php#L1866)) — Isotope init for masonry layout in the editor; bypasses `frontend/element_ready` which doesn't fire in Elementor's iframe.
- **Load More is shared infrastructure** — `load-more.js` is loaded by Post_Grid, Product_Grid, Pro's Filterable_Gallery, and Woo widgets. AJAX endpoint `wp_ajax_load_more` / `wp_ajax_nopriv_load_more` (in [`Ajax_Handler` trait](../../includes/Traits/Ajax_Handler.php#L34)) dispatches by the `class` parameter from the request. The handler forces `post_status = 'publish'` at line 91 and re-asserts it at line 381-383 — protected against draft exposure for standard Post_Grid usage.
- **Render output uses `eael-post-appender` class on the inner container** — both initial render and AJAX-appended content land in this element. Isotope re-layouts via `isotope("appended", $content).isotope("layout")` after each AJAX append.
- **Infinity scroll detection via `.eael-infinity-scroll` class on wrapper** — `load-more.js` binds a global window scroll listener; when wrapper enters viewport (`offset` data-attr distance), triggers the load-more button click ([lines 262-280 of load-more.js](../../src/js/view/load-more.js#L262)).

## Render Output

```html
<div id="eael-post-grid-<widget-id>" class="eael-post-grid-container">
  <div class="eael-post-grid eael-post-appender eael-post-appender-<widget-id>
              eael-post-grid-style-<preset>"
       data-layout-mode="masonry">

    <!-- For each WP_Query post: -->
    <article class="eael-grid-post eael-post-grid-column" data-id="<post-id>">
      <!-- Template output (varies by preset). Typical:
      <div class="eael-entry-thumbnail">
        <img src="..." alt="...">
      </div>
      <div class="eael-entry-wrapper">
        <h2 class="eael-entry-title"><a href="...">Title</a></h2>
        <div class="eael-entry-meta">Author • Date • Categories</div>
        <div class="eael-entry-content">Excerpt...</div>
        <a class="eael-post-elements-readmore-btn" href="...">Read More</a>
      </div>
      -->
    </article>
    …
  </div>
  <div class="clearfix"></div>
</div>

[?] <!-- Load More button or Infinity Scroll wrapper -->
<div class="eael-load-more-button-wrap [eael-infinity-scroll]"
     data-offset="-200">
  <button class="eael-load-more-button"
          data-widget="<widget-id>"
          data-page-id="<page-id>"
          data-class="Essential_Addons_Elementor\Elements\Post_Grid"
          data-args="<url-encoded WP_Query args>"
          data-page="1"
          data-max-page="3"
          data-layout="masonry"
          data-template="...">
    <span class="eael_load_more_text">Load More</span>
  </button>
</div>
```

Notes:

- `data-layout-mode` on the inner container drives Isotope init in JS ([line 6 of post-grid.js](../../src/js/view/post-grid.js#L6)).
- `eael-post-appender` is the AJAX-append target; new content gets appended here.
- `eael-post-grid-style-<preset>` carries the chosen skin name; SCSS scopes preset-specific rules to this class.
- Each post `<article>` carries `data-id="<post-id>"` — used by Load More to build the `post__not_in` exclude list for `orderby = 'rand'` queries (avoids duplicates).
- Load More button `data-args` is the URL-encoded WP_Query args — AJAX handler parses it back; this is the **server-trust boundary** — handler validates the nonce and forces `post_status = 'publish'`.
- `data-class` carries the PHP class name — used by AJAX handler to dispatch to the right render path. ⚠️ Trust boundary: the handler must validate this against an allowlist (currently checks against known class names with `class_exists`).

## Hooks & Filters

Distinguish between **internal code-reuse hooks** (registered by Lite's own Bootstrap) and Pro extension hooks. Post Grid emits only the former; Pro doesn't extend Post Grid specifically.

| Hook | Type | Listener | Purpose |
| ---- | ---- | -------- | ------- |
| `eael/controls/query` | action (emitted, **internal**) | `Bootstrap::query()` in Lite | Inject shared post-query control section ([Helper line 66](../../includes/Traits/Helper.php#L66)) |
| `eael/controls/custom_positioning` | action (emitted, **internal**, fired 3×) | `Bootstrap::custom_positioning()` in Lite | Inject custom-positioning controls per region |
| `eael/controls/read_more_button_style` | action (emitted, **internal**) | `Bootstrap::read_more_button_style()` in Lite | Inject Read More button Style controls |
| `eael/controls/load_more_button_style` | action (emitted, **internal**) | `Bootstrap::load_more_button_style()` in Lite | Inject Load More button Style controls |
| `wp_ajax_load_more` / `wp_ajax_nopriv_load_more` | action (consumed) | `Ajax_Handler::ajax_load_more()` | Shared AJAX endpoint for Load More across Post_Grid, Product_Grid, Pro Filterable Gallery |
| `eael_before_ajax_load_more` | action (emitted) | `Ajax_Handler` | Pre-AJAX hook — Pro's YITH wishlist disables itself here ([line 71 of Ajax_Handler.php](../../includes/Traits/Ajax_Handler.php#L71)) |

⚠️ The four `eael/controls/*` hooks **look like** Pro extension points but they're **Lite-internal code reuse**. Third parties can listen to them, but the primary handlers are in Lite's Bootstrap. Renaming would break Lite, not just Pro.

For shared patterns referenced in this doc, see [`_patterns.md`](_patterns.md).

## JavaScript Lifecycle

- **Triggers (two scripts):**
  - `frontend/element_ready/eael-post-grid.default` → `PostGrid($scope, $)` in `post-grid.js` (Isotope init for masonry layout)
  - `document.on('click', '.eael-load-more-button', …)` (delegation) in `load-more.js` — handles AJAX pagination
  - Global window scroll listener in `load-more.js` — triggers `.eael-load-more-button` click when `.eael-infinity-scroll` wrapper enters viewport
- **Guard:** none for `PostGrid`; delegation pattern in `load-more.js` is idempotent
- **Vendor dependencies:** Isotope (masonry layout), imagesLoaded (defer layout until images load)
- **Reads on init:** `data-layout-mode` on `.eael-post-appender`; per-post `data-id` on each `.eael-grid-post`
- **Isotope init:** `$gallery.isotope({ itemSelector: '.eael-grid-post', layoutMode: 'masonry', percentPosition: true, isOriginLeft: !isRTL })` — RTL-aware
- **AJAX request payload:** `{ action: 'load_more', class, args, page, page_id, widget_id, nonce, template_info }` — plus `post__not_in` for random-order queries
- **AJAX response handling:** appends to `.eael-post-appender`; for masonry layouts, calls `isotope('appended').isotope('layout')` + `imagesLoaded().progress()` for re-flow
- **`post__not_in` for `orderby == 'rand'`** — collects all visible `.eael-grid-post[data-id]` and sends as exclude list to prevent duplicate posts in random-order pagination
- **Editor preview script** — inline `<script>` block in render() output handles Isotope init in the editor iframe (where `frontend/element_ready` doesn't fire reliably)

## Common Issues

### Masonry layout breaks / posts overlap after Load More

- **Likely cause:** Isotope `appended → layout` didn't run, OR images loaded after `layout()` was called (causing height miscalculation)
- **Diagnose:** check console for Isotope errors; verify `imagesLoaded().progress()` is firing after append
- **Fix:** Hard refresh; if persistent, the page may have a JS error stopping `load-more.js` execution — check `localize` global is defined

### Load More button shows on second page even after exhausting posts

- **Likely cause:** server returned content (e.g. a wrapped `<p class="no-posts-found">`) but JS didn't detect the no-posts marker correctly
- **Diagnose:** inspect AJAX response — does it contain `.no-posts-found`?
- **Fix:** the JS check at line 145 of `load-more.js` looks for the class; if AJAX response wraps it unexpectedly, button stays visible — known edge case

### Infinity scroll fires repeatedly without loading new content

- **Likely cause:** scroll handler binds globally and re-fires on every scroll event; if AJAX is slow or fails, scroll trigger repeats
- **Diagnose:** browser console for rapid `POST admin-ajax.php` requests
- **Fix:** `eael-load-more-button` gets `disabled` attribute during request; subsequent triggers no-op via `$this.attr('disabled', true)` ([line 27](../../src/js/view/load-more.js#L27)). If still spamming, check that the disable is reaching the button

### Custom skin / preset doesn't appear in picker

- **Likely cause:** template file not in `includes/Templates/Post-Grid/`; or no matching preview image in `assets/admin/images/layout-previews/` so picker uses placeholder
- **Diagnose:** check `get_template_list_for_dropdown()` (in HelperTrait) — what does it return?
- **Fix:** add template file with correct naming; add preview image `post-grid-<key>.png`

### Read More button text doesn't update after editing

- **Likely cause:** the text is cached in a transient (`eael_post_grid_read_more_button_text_<id>`) for 24 hours ([line 1830 of Post_Grid.php](../../includes/Elements/Post_Grid.php#L1830)); template reads from transient
- **Diagnose:** the widget re-saves the transient on every render — so editing + saving should update; if not, transients table may be cached
- **Fix:** clear object cache; or simply preview the page (re-render sets a fresh transient)

### `post__not_in` parameter overwritten on AJAX load (random order)

- **Likely cause:** `orderby == 'rand'` AJAX sends `post__not_in` of currently-visible post IDs; if user has filters that also use `post__not_in`, JS overwrites the server-side value
- **Diagnose:** by design — random-order requires it; non-random uses simple paging
- **Fix:** known trade-off; switch to a different orderby if your query needs `post__not_in`

### AJAX returns "Permission denied" or empty for non-logged-in users

- **Likely cause:** nonce verification failed — `localize.nonce` is generated server-side and expires after 24h; long-cached pages may have stale nonce
- **Diagnose:** browser console for nonce error response
- **Fix:** invalidate page cache; or use a shorter cache TTL on pages with Post_Grid

## Known Limitations

- **Transient-based settings passing to template files** ([line 1830-1831](../../includes/Elements/Post_Grid.php#L1830)) — fragile pattern: 24-hour transient TTL, key per-widget-id, requires re-render to refresh; alternative would be passing `$settings` as a function argument to a render-callable.
- **Control id typo `excerpt_expanison_indicator`** — `expanison` should be `expansion`. Renaming breaks saved widget data.
- **Hardcoded RTL-aware `isOriginLeft: !isRTL`** in Isotope init — works but `body.rtl` class detection is brittle; would prefer `dir="rtl"` attribute check.
- **Editor preview script duplicates the frontend init** — separate inline `<script>` block at the end of `render()` for the editor. If frontend init logic changes, both need updating.
- **Shared `load-more.js` is widget-class-dispatched** — adding a new pagination-aware widget requires editing `load-more.js` to add a new branch. Brittle; one widget's bug can break all four.
- **`load_more` AJAX endpoint receives URL-encoded WP_Query args from the client** ([line 18 of load-more.js](../../src/js/view/load-more.js#L18) reading `data-args`) — the args originally come from server-side `print_load_more_button()`. Tampering would let an attacker inject query args, but the AJAX handler forces `post_status = 'publish'` ([line 91 of Ajax_Handler.php](../../includes/Traits/Ajax_Handler.php#L91)) preventing draft exposure. Other args (post_type, tax_query) are not validated against the original widget settings — potential information disclosure of public content under different filters than intended.
- **Three CSS files load for one widget** — `load-more.min.css` + `post-grid.min.css` + `product-grid.min.css` ([config.php line 11-24](../../config.php#L11)) — `product-grid.min.css` is loaded even on non-WooCommerce sites because of historical shared styling. Wasted bandwidth on pages without Product_Grid.
- **No `elementStatusCheck` guard** in `post-grid.js` — re-fired `elementor/frontend/init` can re-bind Isotope, but Isotope's `isotope()` call is idempotent so impact is minimal.
- **`get_settings()` not `get_settings_for_display()`** at line 1778 — dynamic-tag values may not be resolved when rendered. Other widgets use `get_settings_for_display()`. Inconsistency.
- **Custom positioning controls fire `do_action` 3 times** ([lines 1152, 1299, 1311](../../includes/Elements/Post_Grid.php#L1152)) with different parameter sets — Bootstrap handler must distinguish via the additional args. Maintenance burden if signature changes.
- **`get_style_depends()` returns `font-awesome-4-shim`** ([line 51](../../includes/Elements/Post_Grid.php#L51)) which is a deprecated handle in modern Elementor versions.
